# CodeRabbit

CodeRabbit CLI を実行して現在の変更をレビューし、指摘された問題を一つずつ修正・コミットする。

## ワークフロー

### 1. レビューの実行

CodeRabbit を prompt-only モードで実行し、レビューのフィードバックを取得する:

```bash
coderabbit review --prompt-only
```

### 2. フィードバックの評価

出力を読み、対処すべき問題（バグ、ロジックエラー、セキュリティ上の懸念、スタイル違反）を特定する。

単なるスタイルの好みや誤検知の指摘は無視する。

### 3. 指摘ごとに修正・コミット

対処すべき指摘それぞれについて:

1. 修正を適用する
2. 変更ファイルを `git add <file>...` でステージングする
3. 修正内容を説明するコミットメッセージでコミットする（プロジェクトのコミット規約に従う）

### 4. 再レビュー

`coderabbit review --prompt-only` を再度実行し、修正の確認と残りの問題の有無をチェックする。

対処すべき問題がなくなるまでステップ 2-4 を繰り返す。

### 5. 報告

行ったすべてのコミットをユーザーに報告する。
