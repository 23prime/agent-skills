# Markdown Writer

プロジェクトで設定されたルールに基づき、`markdownlint-cli2` をエラーゼロで通過する Markdown を書く。

## ルール

### フォーマット

- 順序なしリストにはダッシュ (`-`) を使用する。アスタリスクやプラス記号は使わない (MD004)
- 強調・太字にはアスタリスク (`*`) を使用する。アンダースコアは使わない (MD049, MD050)
- 行の長さ制限なし (MD013 は無効)
- UTF-8 エンコーディング、LF 改行、末尾の空白は除去、最終行に改行を入れる
- Markdown ファイルのインデントは 2 スペース

### HTML

- `<br>` のみインライン HTML として許可 (MD033)
- その他すべての HTML タグは禁止

### テーブル

- テーブルのパイプ周りのスペースを統一する (MD060)
- セパレータ行はダッシュの前後にスペースを入れる: `| --- |`（`|---|` は不可）

### 全般

- 見出しレベルは飛ばさない（例: `##` から `####` への飛ばしは不可）
- 見出し、コードブロック、リストの前後に空行を入れる
- 見出しの末尾に句読点を付けない
- フェンスドコードブロックには言語識別子を指定する

## 検証

Markdown ファイルの作成・編集後、以下を実行する:

```bash
mise run fix-md
mise run check-md
```

自動修正後もエラーが残る場合は、`check-md` がエラーゼロになるまで手動で修正する。
