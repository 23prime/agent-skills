# Introduction

Goは新しい言語である。既存の言語からアイデアを借りているものの、効果的なGoプログラムを他の関連言語で書かれたプログラムとは性質の異なるものにする特異な特性を持っている。C++やJavaのプログラムを単純にGoに翻訳しても、満足のいく結果が得られる可能性は低い。Javaのプログラムは、Goではなく、Javaで書かれているのである。一方、Goの視点から問題について考えることで、成功するが全く異なるプログラムを生み出せる可能性がある。言い換えれば、Goをうまく書くには、その特性とイディオムを理解することが重要である。また、Goでプログラミングする際の確立された慣習(命名、フォーマット、プログラム構成など)を知ることも重要である。そうすることで、あなたが書いたプログラムを他のGoプログラマが理解しやすくなる。

このドキュメントは、明快でイディオム的なGoコードを書くためのヒントを提供する。[言語仕様](https://go.dev/ref/spec)、[Tour of Go](https://go.dev/tour/)、および[How to Write Go Code](https://go.dev/doc/code.html)を補完するものであり、これらはすべて最初に読むべきである。

2022年1月追記: このドキュメントは2009年のGoのリリース時に書かれたものであり、それ以降大幅な更新は行われていない。言語自体の使い方を理解するための優れたガイドではあるが、言語の安定性のおかげで、ライブラリについてはほとんど触れておらず、ビルドシステム、テスト、モジュール、多態性など、このドキュメントが書かれてから起きたGoエコシステムの重要な変更については何も述べていない。更新の計画はない。多くのことが起こり、増え続ける大量のドキュメント、ブログ、書籍が現代的なGoの使い方を見事に説明しているためである。Effective Goは引き続き有用であるが、読者はこれが完全なガイドからは程遠いことを理解すべきである。文脈については[issue 28782](https://go.dev/issue/28782)を参照されたい。
