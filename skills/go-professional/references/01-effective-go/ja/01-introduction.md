# イントロダクション

Goは新しい言語である。既存の言語からアイデアを借用してはいるものの、効果的なGoプログラムを特徴づける性質は、その親戚言語で書かれたプログラムとは異なる独特なものである。C++やJavaのプログラムをGoに直訳しても、満足のいく結果は得られないだろう。Javaプログラムは、Goではなく、Javaで書かれるものである。一方、Goの視点から問題について考えることで、成功した、しかし全く異なるプログラムが生まれる可能性がある。言い換えれば、Goを上手に書くには、その性質とイディオムを理解することが重要である。また、命名規則、フォーマット、プログラム構成など、Goプログラミングにおける確立された慣習を知ることも重要である。そうすれば、あなたが書くプログラムは他のGoプログラマーにとって理解しやすいものになる。

このドキュメントは、明確でイディオマティックなGoコードを書くためのヒントを提供する。これは[言語仕様](https://go.dev/ref/spec)、[Tour of Go](https://go.dev/tour/)、および[How to Write Go Code](https://go.dev/doc/code.html)を補完するものであり、これらはすべて最初に読むべきである。

2022年1月追記: このドキュメントは2009年のGoリリース時に書かれ、それ以降大きな更新は行われていない。言語自体の使い方を理解するための良いガイドではあるが、言語の安定性のおかげで、ライブラリについてはほとんど触れておらず、ビルドシステム、テスト、モジュール、ポリモーフィズムなど、執筆以降のGoエコシステムへの重要な変更については何も触れていない。更新の予定はない。なぜなら、多くのことが起こり、現代のGoの使い方を説明する大量かつ増え続けるドキュメント、ブログ、書籍が素晴らしい仕事をしているからである。Effective Goは依然として有用であるが、読者はこれが完全なガイドからは程遠いことを理解すべきである。詳細は[issue 28782](https://go.dev/issue/28782)を参照のこと。

## 例

[Goパッケージソース](https://go.dev/src/)は、コアライブラリとしてだけでなく、言語の使い方の例としても機能することを意図している。さらに、多くのパッケージには、[go.dev](https://go.dev/)ウェブサイトから直接実行できる、動作する自己完結型の実行可能な例が含まれている。例えば[この例](https://go.dev/pkg/strings/#example-Map)など（必要に応じて、「Example」という単語をクリックして開く）。問題へのアプローチ方法や何かがどのように実装されるかについて疑問がある場合、ライブラリのドキュメント、コード、例が答え、アイデア、背景を提供してくれる。
